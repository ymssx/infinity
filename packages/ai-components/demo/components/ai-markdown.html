<ai-markdown>

AI Markdown 组件 — LLM 生成 Markdown 并渲染为精美 HTML

<ai-markdown> 让 LLM 以 Markdown 格式输出内容，然后自动将其渲染为精美的 HTML。 内置了完整的 Markdown 解析器（零外部依赖），支持标题、列表、表格、代码块、引用等所有常用 Markdown 语法。 非常适合文档、教程、博客文章、技术报告等结构化文本内容。

Markdown 流式渲染 多主题 零依赖 支持 ai-data

💡 为什么选 ai-markdown？ Markdown 是 LLM 最擅长的输出格式。让 LLM 用 Markdown 输出，比直接生成 HTML 更可靠、更结构化， 而且内置的渲染器提供统一的排版和样式，确保视觉效果一致。

快速开始

<!-- 基础用法 -->
<ai-markdown p="写一篇关于 TypeScript 泛型的简短教程"></ai-markdown>

<!-- 暗色主题 -->
<div class="bg-slate-900 p-6 rounded-xl">
  <ai-markdown p="解释 Promise 的工作原理" theme="dark"></ai-markdown>
</div>

<!-- 搭配 ai-data -->
<ai-markdown p="根据这些数据写一份分析报告">
  <ai-data p="mock一组季度销售数据，包含4个季度的收入和利润" />
</ai-markdown>

属性

属性 类型 默认值 说明
p string — 必填。自然语言提示词
theme string "github" 渲染主题。可选 "github"、"minimal"、"dark"
class string — 附加到内容容器的 CSS 类名

主题预览

github 默认

GitHub 风格。清爽、专业，适合技术文档和博客。带有渐变标题底部边框。

<ai-markdown p="..." theme="github"></ai-markdown>
minimal

极简风格。衬线字体，更大的行距，适合阅读舒适度优先的长文本。

<ai-markdown p="..." theme="minimal"></ai-markdown>
dark

暗色主题。适合深色背景。代码块使用更深的底色，内联代码为粉色。

<ai-markdown p="..." theme="dark"></ai-markdown>

支持的 Markdown 语法

文本格式

  • ✅ 标题 # ~ ######
  • ✅ 粗体 **bold**
  • ✅ 斜体 *italic*
  • ✅ 删除线 ~~text~~
  • ✅ 行内代码 `code`
  • ✅ 链接 [text](url)
  • ✅ 图片 ![alt](url)

块级元素

  • ✅ 代码块（带语言高亮类名）
  • ✅ 无序列表 - item
  • ✅ 有序列表 1. item
  • ✅ 引用 > text
  • ✅ 表格 | col |
  • ✅ 水平线 ---
  • ✅ 段落（自动识别换行）

使用示例

技术教程

LLM 擅长以 Markdown 格式输出结构化教程。

<ai-markdown p="写一篇简短的 CSS Grid 入门教程，包含代码示例"></ai-markdown>

对比表格

非常适合让 LLM 生成对比分析表格。

<ai-markdown p="用表格对比 React, Vue, Svelte 三个框架的优缺点、学习曲线和社区生态"></ai-markdown>

数据报告

搭配 <ai-data> 获取数据后生成分析报告。

<ai-markdown p="根据数据写一份月度销售分析报告，包含趋势分析和建议">
  <ai-data p="mock一组电商月度销售数据，包含各品类的销售额和环比增长率" />
</ai-markdown>

暗色主题

在暗色背景中使用 theme="dark"。

<div class="bg-slate-900 p-8 rounded-xl">
  <ai-markdown p="用 Markdown 解释 async/await 的工作原理，包含代码示例" theme="dark"></ai-markdown>
</div>

代码审查

将代码片段作为数据输入，让 LLM 写 code review。

<ai-markdown p="对这段代码进行 code review，指出问题并给出改进建议">
  <ai-data p="生成一段有常见问题的 JavaScript 代码（如变量命名不规范、缺少错误处理等）" format="text" />
</ai-markdown>

工作原理

1
等待数据 — 如果内部有 <ai-data>，先等待解析完成。
2
Markdown 系统提示词 — 使用专用提示词，指示 LLM 以纯 Markdown 格式输出。
3
流式累积 — 每次收到 chunk，追加到 Markdown 缓冲区。
4
实时渲染 — 每次 chunk 到达时，对完整缓冲区重新解析 Markdown → HTML → innerHTML 渲染。用户看到内容逐步丰富。
5
主题注入 — 根据 theme 属性注入对应的 CSS 样式表，所有 Markdown 元素自动获得统一样式。
